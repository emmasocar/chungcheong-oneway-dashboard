제로편도 서비스

편도 반납 차량의 역방향 매칭으로 탁송비 절감 + 추가 매출 확보

왜 해야 하는가

고객 관점의 문제

  • 편도 수수료 과도 - 편도 이용 시 별도 수수료 부과로 비용 부담
  • 수요 이탈 - 비용 부담으로 편도 이용 자체를 포기하는 잠재 수요 존재

회사(쏘카) 관점의 문제

  • 탁송비 연 60.6억원 - 편도 반납 후 차량 원위치 비용 (전국, 제주 제외)
  • 탁송 중 매출 0원 - 핸들러 이동 중 차량 비가동, 수익 제로
  • GP 직접 감소 - 탁송비는 순비용, 매출 없이 비용만 발생

해결 방안: 롤링 재고 + Push 알림

A고객이 편도 반납한 차량을, 역방향 B고객에게 편도수수료 0원으로 제공.
B고객은 출발 지역 내 원하는 곳에 반납하면 되고, 쏘카는 지역 내 재배치(중위 ~10,000원)만 부담.

1
A고객: 대전 X존 → 청주 편도 반납 (정상 요금 수취)
매출: 대여+주행+편도수수료
2
청주 허브존에 24h 대기 + 역방향 고객에게 "청주 출발 편도 0원" push 알림
대기 비용: ~9.5K/일 (AS-IS에서도 동일 발생)
3
B고객: 청주 → 대전 내 원하는 곳에 반납 (편도수수료 0원, 대여+주행만 지불)
매출: 대여+주행
4
쏘카: 반납지 → 차량 소속존 지역 내 재배치 (같은 시도 내)
비용: ~10,000원
5
매칭 실패 시: 기존 유료편도 프로세스 진행 → 추가 손실 0원 (Zero-Risk)
동일 비용

핵심: B고객은 출발 지역 내 원하는 곳 어디든 자유 반납

B고객이 A고객의 출발존(X존)으로 정확히 돌아갈 필요 없음. 같은 시도 내 어떤 쏘카존이든 OK
쏘카는 반납지→차량 소속존 재배치만 수행 (중위 ~10,000원) → 쏘카존 반납 허들 제거, 고객 편의성 극대화
* 지역 내 재배치 비용 = 같은 시도 내 편도 중위 탁송비 기준 (대전 9.8K / 충북 11.2K / 전국 평균 ~11K)

왜 D-7 사전 매칭이 아닌가?

전국 편도 예약의 54%가 출발 3시간 이내, 81%가 당일 예약
D-7 시점에는 IN 차량 도착 여부 자체를 알 수 없음. 7일+ 사전 예약은 5.1%에 불과
→ 사전 매칭 대신 반납 후 대기 → 실시간 push 알림 방식이 현실적

고객 여정: B고객이 매칭 차량을 잡는 과정

핸들러 앱과 유사하게, B고객이 직접 매칭 가능 차량을 확인하고 선택하는 구조.
D-7 사전 매칭이 아니므로 실제 반납된 차량의 차명(car_name), 출발지, 방면, 출발 가능 시간이 모두 노출됨.

B고객 여정

"청주 출발 → 대전 방면 편도수수료 0원 차량이 있습니다"
2

차명 / 출발지 / 도착 방면 / 출발 가능 시간

편도수수료 0원, 대여료+주행료만 결제
4

쏘카존 아닌 곳도 OK, 시도 내 자유 반납

D-7이 아니기 때문에 가능한 것

D-7 사전 매칭이었다면 아직 반납 전이라 차종만 노출 가능 (어떤 차가 올지 불확실)
롤링 재고 모델은 이미 반납된 차량이므로 → 차명, 위치, 출발 가능 시간 모두 확정
고객 입장에서 "내가 탈 차"를 직접 보고 고르는 핸들러앱 방식의 확실한 UX

데이터 근거

본 대시보드의 모든 수치는 2025년 실데이터 기반입니다. 각 수치의 산출 쿼리를 아래에서 확인할 수 있습니다.

데이터 소스: socar-data BigQuery
기준 기간: 2025-01-01 ~ 2025-12-31
필터: state=3, member_imaginary IN (0,9), way IN ('z2d_oneway','d2d_oneway'), 제주 제외
테이블: reservation_v2 + reservation_dtod_info + carzone_info + profit_socar_reservation
전국 편도 총건수 + 탁송비 → 377,506건, 60.6억 
SELECT
  COUNT(*) AS cnt,
  ROUND(SUM(
    COALESCE(transport_cost_socar,0) + COALESCE(transport_cost_d2d,0)
    + COALESCE(transport_cost_z2d,0) + COALESCE(transport_cost_mobility,0)
  )) AS total_transport
FROM reservation_v2 r
LEFT JOIN carzone_info cz ON r.zone_id = cz.id
WHERE date >= '2025-01-01' AND date < '2026-01-01'
  AND state = 3 AND member_imaginary IN (0, 9)
  AND way IN ('z2d_oneway','d2d_oneway')
  AND cz.region1 != '제주특별자치도'
양방향 매칭 KPI (전국 38,765쌍 / 충청 1,458쌍) 
-- 양방향 매칭: 같은 구간 A→B, B→A 중 적은 쪽(MIN) = 매칭 가능 쌍
-- 개별 건수: 쌍 × 2, 50% 매칭 시 = 쌍 수와 동일
-- 탁송비 절감: med_transport_A + med_transport_B - 20,000 (재배치 비용)
-- 추가 매출/GP: (avg_A + avg_B) / 2 (양방향 평균)

WITH agg AS (
  SELECT start_region, end_region,
    COUNT(*) AS cnt,
    APPROX_QUANTILES(transport_cost, 100)[OFFSET(50)] AS med_transport,
    AVG(core_rev) AS avg_rev, AVG(gp) AS avg_gp
  FROM cross_region_oneway
  GROUP BY 1, 2
)
SELECT
  LEAST(a.cnt, b.cnt) AS match_pairs,
  a.med_transport + b.med_transport - 20000 AS pair_transport_saving,
  (a.avg_rev + b.avg_rev) / 2.0 AS pair_avg_rev,
  (a.avg_gp + b.avg_gp) / 2.0 AS pair_avg_gp
FROM agg a
JOIN agg b ON a.start_region = b.end_region
  AND a.end_region = b.start_region
WHERE a.start_region < a.end_region
AS-IS 건당 비용 → 탁송 16,087 + 주차 1,668 + 기회비용 7,854 = 25,607원 
-- 탁송비: transport_cost 4개 컬럼 합산 평균 = 16,087원
-- 주차비: parking_cost_d2d + parking_cost_z2d 평균 = 1,668원
--         (parking_cost_oneway는 전부 0)
-- 기회비용: 차량 일평균 GP 7,854원 × 유휴 24.8h(≈1일)
-- 유휴시간: 편도 return_at ~ 같은 car_id 다음 예약 start_at
--           평균 24.8h, 중위 17h, 24h이상 31.3%
지역 내 재배치 비용 → 중위 ~10,000원 
-- 같은 시도 내 편도 탁송비 중위값
-- 대전: 9,825원 / 세종: 9,839원 / 충남: 10,412원 / 충북: 11,197원
-- 서울: 10,977원 / 경기: 13,015원 / 부산: 10,708원

SELECT start_region,
  APPROX_QUANTILES(transport_cost, 100)[OFFSET(50)] AS med_transport
FROM oneway
WHERE start_region = end_region  -- 같은 시도 내 이동
GROUP BY start_region
24시간 매칭 윈도우 ROI → 3.0배 
-- 대기 비용: 주차비 1,668 + 기회비용 7,854 = 9,520원
--   (AS-IS에서도 평균 24.8시간 유휴 → 추가 비용 아님)
-- 매칭 시 건당 GP 개선: 28,124원 (10.9억 ÷ 38,766건)
-- ROI: 28,124 ÷ 9,520 = 3.0배
-- 매칭 실패 시: 기존 탁송 프로세스, 추가 손실 = 0
